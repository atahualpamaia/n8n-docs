Active Webhook POST — Fireflies notification on call completion

(Fireflies) PID Call Analysis + Coach

Automatically analyzes every sales discovery call recorded via Fireflies.ai. Fetches the full transcript, extracts Pain/Impact/Decision signals, scores each dimension with GPT-4o-mini, saves results to Airtable, and emails the rep a PID report plus a personalized coaching message.

Execution Flow

1
Trigger & Fetch

Fireflies sends a webhook on call completion. The workflow fetches the full transcript via the Fireflies node.

2
Input Cleaning

Clean Input + Rep Map normalizes the transcript and maps the rep email to their display name and nick (14 reps mapped).

3
PID Extraction

Two parallel GPT calls run: PID Extraction pulls Pain, Impact, and Decision signals as structured JSON. Call Intelligence classifies the call type and extracts prospect metadata.

4
Scoring — 3 parallel GPT calls

PID Score: Pain, PID Score: Impact, and PID Score: Decision each score their dimension on a 0–10 scale with criteria-level reasoning.

5
Calculate & Save

Calculate Scores merges the three scores into a normalized total. Airtable: Upsert Call saves the full record. Two emails go out: PID Report (scores + evidence) and PID Coaching (personalized improvement tips).

Save

Calculate Score merges scores. Airtable: Upsert Analysis saves the full record. Short calls: Rapport Analysis runs instead and saves to Airtable: Upsert Short Call.

Active Webhook POST /send-user-data — Snov.io registration

Inbound Leads Qualification

Processes every new Snov.io registration. Verifies email (polling loop), phone (Abstract API), person name, and company name (GPT). Applies 5 ICP gates and scores the lead. Routes qualified leads to Pipedrive (Person + Org + Deal) and sends them to Clay for enrichment. All records are persisted in Airtable.

Execution Flow

1
Trigger & Raw Record

Webhook receives the registration payload. Airtable: Create Registration immediately creates a raw record.

2
Verification — 6 parallel

Email: Snov.io API with polling loop (up to 8×15s). Person name: GPT validation. Company name: GPT validation. Domain match: GPT check. Phone: Abstract API. Direct passthrough.

3
Record Assembly & ICP Gates — 5 parallel

Build Lead Record assembles all results, assigns team, calculates lead score and data score. Five gates: country, personal email, job position, main goals, data score (must be 100). All must pass.

4
Pipedrive Creation — 2 parallel

Search Person → update or create. Create Organization (always). Both merge before deal creation.

5
Deal + Enrichment

Create Deal with 16 custom fields. Save Deal ID to Airtable. Send to Clay (with retry). Check for org duplicates.

Active Calendly webhook — Discovery Call booked

Create/Update a deal from Discovery Call

When a prospect books a Discovery Call via Calendly, this workflow creates or updates their deal in Pipedrive. Verifies email and phone, searches for an existing person, creates person and org if needed, creates or updates the deal, assigns the correct owner, adds a deal note, and links the Calendly activity to the deal.

Execution Flow

1
Trigger & Parse

Calendly sends a booking webhook. Parse Calendly Event extracts attendee data. Add Deal Owner ID maps the host to the correct Pipedrive user ID.

2
Person Lookup

Searches Pipedrive for an existing person by email. If found, checks for an open deal. If not found, verifies phone and creates the person.

3
Email Verification

Snov.io email verification polling loop — same as Inbound Leads Qualification.

4
Deal Create / Update

Creates or updates the deal with owner, stage, and custom fields. Changes owner and adds a deal note.

5
Activity Linking

Waits for the Calendly activity to appear in Pipedrive. Finds the activity by type. Links it to the deal.

6
Airtable Sync

Creates or updates the Airtable registration record with deal data.

Active Pipedrive deal update trigger — real-time

Sync CRM with Airtable + Add Time per Stage

Fires on every Pipedrive deal update. Filters to SALES 2026 pipeline only. Syncs deal data to Airtable and tracks time spent in each funnel stage (Registration, MQL Trying, MQL Contact, SQL, SAL Trial, SAL Commit). Marks deal milestones — MQL, SAL, First Contact, Won, Lost — in Airtable.

Execution Flow

1
Trigger & Filter

Pipedrive Trigger fires on any deal change. SALES 2026 only? gate filters out all other pipelines.

2
Build & Sync

Build Deal Data assembles the normalized record. Update Sales Database upserts it to Airtable.

3
Stage Time Tracking — 6 parallel

For each stage the deal may have exited, queries Pipedrive deal history to calculate time spent. Runs only if the deal was previously in that stage.

4
Milestone Marking

Checks if the deal entered MQL, SAL, First Contact, Won, or Lost. Updates Airtable with the milestone timestamp.
