transcript-embedding-generator-daily

This workflow generates vector embeddings for call transcripts stored in Supabase that do not yet have embeddings. It queries for unembedded records, batches them, sends them to OpenAI text-embedding-3-small for vectorization, and updates each record with its embedding. This powers semantic search across call transcripts in the Sales Intelligence dashboard.

Execution Flow
1
Fetch Unembedded Records — Queries Supabase for call_transcripts rows where the embedding column is null, up to 50 records per run.
2
Batch & Embed — Prepares transcript text into a batch payload and sends it to OpenAI Embeddings API (text-embedding-3-small, 1536 dimensions).
3
Update Records — Maps each embedding back to its source record and updates the Supabase call_transcripts table with the generated vector.

(API4COM) Phone Call Analysis (Supabase)

This workflow automates the analysis of phone call recordings by transcribing audio, extracting conversational intelligence, scoring various competencies, and syncing results to Supabase. It supports both full-length and short call workflows, ensuring all call data and insights are archived for future reference and analytics. The process enhances call review efficiency by leveraging AI for sentiment and score extraction while maintaining strong data consistency with Supabase.

Execution Flow
1
Input & Preparation — Receives the webhook event, cleans and maps input data, and prepares the activity record.
2
Audio Processing & Transcription — Downloads call audio and transcribes both full-length and short calls as needed.
3
AI Analysis & Scoring — Extracts pain points, call intelligence, rapport, and scores multiple call aspects using OpenAI.
4
Data Mapping & Storage — Maps analysis results and upserts activity, analysis, and transcript records to Supabase.
Technical Notes

Regularly monitor OpenAI API output formats for changes, and review Supabase integration for schema compatibility after updates.
