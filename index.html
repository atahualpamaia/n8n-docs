ActiveWebhook POST — Fireflies call completion

(Fireflies) PID Call Analysis + Coach

Analyzes every discovery call recorded via Fireflies.ai. Fetches the transcript, extracts Pain/Impact/Decision signals, scores each dimension with GPT-4o-mini, saves to Airtable, and emails the rep a PID report and personalized coaching message.

Execution Flow
1
Trigger & Fetch — Fireflies sends a webhook on call completion. Workflow fetches the full transcript via the Fireflies node.
2
Input Cleaning — Clean Input + Rep Map normalizes the transcript and maps the rep email to display name/nick. 14 reps mapped.
3
PID Extraction — Two parallel GPT calls: PID Extraction pulls Pain, Impact, Decision as structured JSON. Call Intelligence classifies call type and extracts prospect metadata.
4
Scoring — 3 parallel GPT calls — PID Score: Pain, Impact, and Decision each score 0–10 with criteria-level reasoning.
5
Calculate & Save — Calculate Scores merges three scores into a normalized total. Saves to Airtable. Sends PID Report + PID Coaching emails to rep.
Technical Notes

Rep map is hardcoded in Clean Input node — update when team changes. Internal call filter uses @snov.io domain check. continueOnFail enabled on all GPT nodes.

ActiveWebhook POST /send-user-data — Snov.io registration

Inbound Leads Qualification

Processes every new Snov.io registration. Verifies email (polling loop), phone (Abstract API), name and company (GPT). Applies 5 ICP gates. Routes qualified leads to Pipedrive (Person + Org + Deal) and sends to Clay for enrichment. All records persisted in Airtable.

Execution Flow
1
Trigger & Raw Record — Webhook receives registration. Airtable: Create Registration immediately creates a raw record.
2
Verification — 6 parallel — Email: Snov.io polling loop (up to 8×15s). Person name, company name, domain match: GPT. Phone: Abstract API. Direct passthrough.
3
Record Assembly & ICP Gates — 5 parallel — Build Lead Record assembles all results. Five gates: country, personal email, job position, main goals, data score. All must pass.
4
Pipedrive Creation — 2 parallel — Search Person → update or create. Create Organization. Both merge before deal creation.
5
Deal + Enrichment — Create Deal with 16 custom fields. Save Deal ID to Airtable. Send to Clay with retry. Check org duplicates.
Technical Notes

Data score gate: must equal 100 (name 24 + company 24 + phone 52). Volume: ~40–50K registrations/month. Country blocklist and freemail list hardcoded in code nodes.

Create/Update a deal from Discovery Call

When a prospect books a Discovery Call via Calendly, creates or updates their deal in Pipedrive. Verifies email and phone, searches for existing person, creates person/org if needed, assigns owner, adds deal note, and links the Calendly activity to the deal.

1
Trigger & Parse — Calendly webhook. Parse Calendly Event extracts attendee data. Add Deal Owner ID maps host to Pipedrive user ID.
2
Person Lookup — Searches Pipedrive by email. If found, checks for open deal. If not, verifies phone and creates person.
3
Email Verification — Snov.io polling loop — same as Inbound Leads Qualification.
4
Deal Create / Update — Creates or updates deal with owner, stage, custom fields. Adds deal note.
5
Activity Linking — Waits for Calendly activity to appear. Finds by type. Links to deal.
6
Airtable Sync — Creates or updates Airtable registration record.
Technical Notes

Owner mapping hardcoded in Add Deal Owner ID — update when team changes. Activity linking has a wait node to account for Calendly sync delay.

ActivePipedrive deal update trigger — real-time

Sync CRM with Airtable + Add Time per Stage

Fires on every Pipedrive deal update. Filters to SALES 2026 pipeline. Syncs deal data to Airtable and tracks time spent in each funnel stage. Marks deal milestones — MQL, SAL, First Contact, Won, Lost — in Airtable.

Execution Flow
1
Trigger & Filter — Pipedrive Trigger fires on any deal change. SALES 2026 only? gate filters other pipelines.
2
Build & Sync — Build Deal Data assembles normalized record. Update Sales Database upserts to Airtable.
3
Stage Time Tracking — 6 parallel — For each stage the deal may have exited, queries Pipedrive history to calculate time spent.
4
Milestone Marking — Checks if deal entered MQL, SAL, First Contact, Won, or Lost. Updates Airtable with timestamp.
Technical Notes

Stage IDs documented in the Stage IDs sticky note inside the workflow. Pipeline filter must be updated if pipeline ID changes.
